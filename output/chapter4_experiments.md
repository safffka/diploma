# 4. Экспериментальное исследование

## 4.1 Экспериментальная установка

**Набор данных.** Эксперименты проведены на наборе данных ISPRS Vaihingen — эталонном наборе данных для семантической сегментации аэрофотоснимков, предоставленном Международным обществом фотограмметрии и дистанционного зондирования (ISPRS). Набор содержит 16 аннотированных ортофотоизображений сверхвысокого разрешения (GSD ≈ 9 см) города Вайхинген (Германия). Каждый пиксель размечен в один из шести семантических классов: непроницаемые поверхности (impervious surface), здания (building), низкая растительность (low vegetation), деревья (tree), автомобили (car), фон/помехи (background/clutter).

**Предобработка данных.** Исходные изображения размером порядка 2000×2000 пикселей разрезаны на тайлы 512×512 с перекрытием 64 пикселя для обеспечения контекста на границах. Разделение на выборки выполнено на уровне полных изображений (а не тайлов) для исключения утечки данных: 11 изображений для обучения, 2 для валидации, 3 для тестирования. Итоговое количество тайлов: обучающая выборка — 220, валидационная — 38, тестовая — 50.

**Аппаратное обеспечение.** Обучение выполнено на Apple M-series с использованием фреймворка MPS (Metal Performance Shaders) для GPU-ускорения на процессорах Apple Silicon.

**Параметры обучения.** Все семь моделей обучены в идентичных условиях для обеспечения корректности сравнения:
- Оптимизатор: AdamW с начальной скоростью обучения 1×10⁻⁴ для моделей, обучаемых с нуля, и 5×10⁻⁵ для моделей с предобученным энкодером;
- Размер мини-пакета: 2;
- Максимальное число эпох: 100;
- Ранняя остановка: patience = 15 эпох без улучшения val_mIoU;
- Функция потерь: комбинация CrossEntropy (вес 0,5) и Dice Loss (вес 0,5) с весами классов, рассчитанными по обратной частоте из статистики обучающей выборки.

**Метрики оценки.** Качество сегментации оценивалось по следующим метрикам:
- mIoU (mean Intersection over Union) — среднее пересечение по объединению по всем классам, основная метрика;
- Boundary IoU (BIoU) — IoU, вычисленный в полосе шириной 2 пикселя вдоль границ объектов, оценивает качество деления границ;
- Параметры модели (Params, млн) — количество обучаемых параметров;
- FLOPs (G) — количество операций с плавающей точкой для одного прямого прохода на входе 512×512.

## 4.4 Проверка гипотез

**Гипотеза H1: mIoU модели с вниманием превышает mIoU базового U-Net на 5 процентных пунктов.**

Результат: **ОТВЕРГНУТА.** Для моделей, обученных с нуля: mIoU(Attention U-Net) = 0,5437, mIoU(U-Net) = 0,5575, дельта = −0,0138 (−1,38 п.п.). Для моделей с предобученным энкодером: mIoU(Attention U-Net PT) = 0,6143, mIoU(U-Net PT) = 0,6182, дельта = −0,0039 (−0,39 п.п.). В обоих случаях модель с вниманием не только не превзошла базовый U-Net на 5 п.п., но показала несколько более низкий результат.

Интерпретация: механизмы внимания (Attention Gate + MHSA) не обеспечивают улучшения общей метрики mIoU в условиях ограниченного объёма данных. Возможные причины: (1) при 220 обучающих тайлах модель с дополнительными параметрами внимания склонна к переобучению; (2) MHSA в боттлнеке 32×32 = 1024 токена имеет ограниченное рецептивное поле по сравнению с масштабом объектов (здания, дороги); (3) вентили внимания перераспределяют, но не создают новую информацию в skip-соединениях.

**Гипотеза H2: вычислительная сложность модели с вниманием не превышает 120% от базового U-Net.**

Результат: **ПОДТВЕРЖДЕНА.** Для моделей с нуля: FLOPs(Attention U-Net) / FLOPs(U-Net) = 302,8 / 298,4 = 1,01 (101%). Для моделей с предобученным энкодером: 83,4 / 75,9 = 1,10 (110%). В обоих случаях соотношение не превышает 1,20. Вычислительные накладные расходы механизмов внимания минимальны: Attention Gate добавляет лишь две свёртки 1×1 и одну sigmoid, а MHSA в боттлнеке работает на малом разрешении (1024 токена).

**Гипотеза H3: Boundary IoU модели с вниманием превышает Boundary IoU базового U-Net на 15 процентных пунктов.**

Результат: **ОТВЕРГНУТА.** Для моделей с нуля: BIoU(Attention U-Net) = 0,3217, BIoU(U-Net) = 0,3154, дельта = +0,0063 (+0,63 п.п.). Для моделей с предобученным энкодером: BIoU(Attention U-Net PT) = 0,3468, BIoU(U-Net PT) = 0,3396, дельта = +0,0072 (+0,72 п.п.). Улучшение составляет менее 1 п.п. вместо ожидаемых 15 п.п.

Интерпретация: вентили внимания действительно улучшают качество границ, но эффект скромный (+0,6–0,7 п.п.). Гипотеза о 15 п.п. улучшения была чрезмерно оптимистичной. Причина: Boundary IoU определяется не столько механизмом агрегации skip-соединений, сколько разрешением признаков на последнем уровне декодера и качеством низкоуровневых текстурных признаков.

Отвержение гипотез H1 и H3 является валидным научным результатом. Оно демонстрирует, что эффективность механизмов внимания существенно зависит от объёма обучающих данных и что на малых выборках (220 тайлов) качество инициализации весов (предобучение) значительно важнее архитектурных усложнений.

## 4.5 Анализ результатов

Проведённый эксперимент позволяет выделить несколько ключевых находок.

**Доминирующая роль предобученного энкодера.** Перенос обучения с ImageNet обеспечивает прирост +6,06 п.п. mIoU (с 0,5575 до 0,6182 для U-Net). Это наибольший фактор улучшения среди всех исследованных, превышающий вклад любого архитектурного решения. При ограниченном объёме данных (220 тайлов) качество инициализации весов оказывается критически важным.

**Минимальное влияние архитектуры декодера.** Разница между U-Net и Attention U-Net составляет −1,38 п.п. mIoU (scratch) и −0,39 п.п. (pretrained). Добавление вентилей внимания и самовнимания в боттлнеке не приводит к статистически значимому улучшению общего качества сегментации. Это указывает на то, что узким местом является не агрегация skip-соединений, а качество извлекаемых энкодером признаков.

**Улучшение качества границ.** Несмотря на отвержение H3, наблюдается устойчивое улучшение Boundary IoU моделями с вниманием: +0,63 п.п. (scratch), +0,72 п.п. (pretrained). Attention Gate выполняет свою функцию — подавляет нерелевантные skip-признаки вблизи границ — но эффект на интегральную метрику невелик.

**Эффективность SegFormer.** При минимальных вычислительных затратах (2,3 млн параметров, 2,9 GFLOPs — в 100 раз меньше U-Net) SegFormer показывает mIoU 0,4585, сопоставимый с DeepLabV3+ (0,4589). Пространственная редукция в механизме внимания (`EfficientSelfAttention` с sr_ratio) и MixFFN обеспечивают эффективное кодирование при минимальной сложности. Модель представляет интерес для применений с ограниченными вычислительными ресурсами.

**FCN как неожиданно сильный базовый метод.** Простейшая модель FCN (3,7 млн параметров) достигает mIoU 0,5322, уступая U-Net лишь 2,5 п.п. Это объясняется тем, что при bilинейной интерполяции до полного разрешения на плотных сценах (аэрофото) пространственное качество определяется преимущественно разрешением контекстных признаков, а не декодером.

**Ранжирование моделей по mIoU:** U-Net PT (0,6182) > Attention U-Net PT (0,6143) > U-Net (0,5575) > Attention U-Net (0,5437) > FCN (0,5322) > DeepLabV3+ (0,4589) > SegFormer (0,4585).

**Ранжирование по эффективности (mIoU / GFLOPs):** SegFormer (0,158 / GFLOP) > Attention U-Net PT (0,0074) > U-Net PT (0,0081) > DeepLabV3+ (0,0138) > FCN (0,0073) > Attention U-Net (0,0018) > U-Net (0,0019).
